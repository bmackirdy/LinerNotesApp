Liner Notes – Workspace Rules
General Principles

The workspace should reflect clarity, consistency, and collaboration mirroring the clean, focused nature of the Liner Notes app itself.

Every team member is responsible for maintaining code quality, documenting changes, and respecting project structure.

Treat this repository as production-grade from day one follow version control discipline and best practices.

Repository Structure
/client/        → React frontend (Vite build system)
/api/           → Express backend
/supabase/      → SQL migrations, seeds, and schema definitions
/docs/          → PRD, Tech Stack, API specs, site map, deployment notes
README.md
.gitignore


Rules:

Do not modify the directory structure without approval from the project lead.

All new files must be created inside the appropriate directory.

Use clear, consistent naming conventions for files and commits.
